# AI Agents

## Principle

Agents produce **artifacts**, not chat. Invisible in UX except when delivering output.

## The 4 agents

### 1. Scenario Narrative Agent (Kimi K2.6)
- **Trigger**: `runScenarioAction()` Server Action (`src/app/(product)/scenario-lab/actions.ts`)
- **Input**: scenario_run JSON (assumptions + computed outputs)
- **Output schema**:
  ```ts
  {
    narrative: string;          // max 500 chars
    risk_warning: string;       // max 200 chars
    confidence: "low" | "medium" | "high";
    key_drivers: string[];      // max 3
  }
  ```
- **Guardrails**:
  - Forbidden words: "guarantee", "promise", "certain", "will deliver", "risk-free"
  - Must mention ≥ 1 assumption
  - If confidence = low → must say so explicitly

### 2. Mining Health Agent (Kimi K2.6)
- **Trigger**: daily 08:00 UTC cron + on-demand from Dashboard
- **Input**: 30d hashprice, difficulty, BTC price, energy cost, uptime, allocation
- **Output schema**:
  ```ts
  {
    alert_level: "green" | "orange" | "red";
    summary: string;            // max 400 chars
    recommendation: string;     // max 200 chars (suggests, never executes)
    key_metrics: {
      mining_margin_score: number;
      hashprice_trend_pct: number;
      operational_confidence: number;
    };
  }
  ```
- **Guardrails**:
  - Red alert → notify manager (no auto-action)
  - Summary must cite ≥ 2 numeric metrics

### 3. Risk Explanation Agent (Kimi K2.6)
- **Trigger**: daily + on every rebalancing event
- **Input**: 5 risk scores + 30d evolution + current allocation
- **Output schema**:
  ```ts
  {
    top_risks: Array<{
      risk_type: "market" | "mining" | "liquidity" | "smart_contract" | "counterparty";
      score: number;
      trend: "rising" | "stable" | "falling";
      explanation: string;       // max 300 chars
      suggested_guardrail: string; // max 200 chars
    }>;
  }
  ```
- **Guardrails**:
  - Guardrails proposed must stay within vault bounds
  - Never claims "we will fix" — proposes, doesn't decide

### 4. Investor Memo Agent (Kimi K2.6, max_tokens 4096)
- **Trigger**: click "Generate memo" + `client_name` + `logo_url`
- **Input**: vault state + 3 scenarios + 3 backtests + methodology version
- **Output schema**:
  ```ts
  {
    sections: {
      executive_summary: string;     // markdown
      yield_source: string;          // markdown
      stress_scenarios: PtaiBlock[]; // structured
      vault_response: string;
      methodology: string;
      trust: string;
      risks_and_terms: string;
    };
    methodology_version: string;
    generated_at: string;            // ISO datetime
    warnings: string[];              // missing data, low-confidence flags
  }
  ```
- **Guardrails**:
  - Methodology + glossary + disclaimers injected as immutable system blocks (Kimi has no native prompt caching; `cache_control` accepted but ignored)
  - Disclaimers NEVER generated, only templated
  - If data missing → explicitly flagged in `warnings[]`, no invention

## Stack

- **Kimi K2.6 via Hypercli** (OpenAI-compatible endpoint) — single model for all 4 agents (ADR-007, 2026-05-26)
- `openai@6.x` SDK pointed at `HYPERCLI_BASE_URL` (key in `HYPERCLI_API_KEY`)
- No native prompt caching (Kimi limitation); `cache_control` is accepted forward-compat but ignored
- All outputs validated via Zod schemas (`src/lib/agents/schemas.ts`)
- Linter strips/rejects forbidden words (`src/lib/agents/validators.ts`)
- Cost/usage tracked in `LlmRun` Prisma table (Kimi pricing $0.60 / $2.50 per MTOK indicative)

## Forbidden absolutely

- ❌ Chat ("Ask the AI" widget)
- ❌ Unsourced recommendations
- ❌ Yield affirmations / guarantees
- ❌ Autonomous decisions (agents propose, humans decide)
