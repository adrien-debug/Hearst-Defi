## Scope

## Yield projection (forward 12m)

```
projected_apy = Σ over buckets b of:
  current_allocation_pct(b) × forward_yield_assumption(b)
```

Forward yield assumptions:
- **Mining**: `net_distributable_mining_yield` from current `hashprice × hashrate × revenue_share − costs`. Annualised using last 30d.
- **USDC base**: weighted average of T-bills + lending APY observed last 30d.
- **BTC tactical**: assumed P&L = 0 in base case (variable, computed by tactical rules under each scenario).
- **Stable reserve**: USDC native yield (~4.5% conservative).

## APY range (low / high)

```
apy_low  = projected_apy × (1 − assumption_risk_factor)
apy_high = projected_apy × (1 + assumption_upside_factor)
```

Where:
- `assumption_risk_factor ∈ [0.10, 0.30]` increasing with timeframe and decreasing data freshness.
- `assumption_upside_factor ∈ [0.10, 0.25]` symmetric.

**No single-point APY is ever published.**

## Stressed APY

Combined scenario: BTC −40% + Hashprice −30% + Mining margin compression.
Computed via the same engine under Bear preset assumptions.

## Confidence score

| Factor | Weight | Definition |
|---|---|---|
| Data freshness | 40% | All sources < 24h old → 100; degrades linearly |
| Historical fit | 30% | Are the input values within the last 36m historical distribution? |
| Out-of-range assumptions | 20% | Count of variables outside historical P5–P95 range |
| Rebalancing room | 10% | Does the vault have allocation headroom within bounds to apply the response? |

Output:
- High: all green
- Medium: 1-2 yellow
- Low: any red → narrative MUST explicitly state low confidence

## Forbidden in any projection output

- Numeric promises ("will deliver 12%")
- "Risk-free" claims
- Sharpe / Sortino / VaR in primary output (Advanced mode only)
- Probability without explicit interval and methodology link
